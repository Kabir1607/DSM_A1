Disaster Evolution & Response Pipeline: Intelligence Brief

1. Technical Challenges & Methodology

Building a robust data pipeline to extract deep-tier disaster intelligence presented several technical hurdles, primarily concerning data availability, structural inconsistencies across APIs, and anti-scraping measures. 

The first major challenge involved extracting the "Impact" table data (Population Exposure and Vulnerability). While GDACS provides a web interface, relying on HTML scraping for these specific tables proved brittle due to dynamic rendering. Instead, we pivoted to the GDACS CAP (Common Alerting Protocol) XML feed. By parsing the `<parameter>` tags within the XML namespaces, we reliably extracted consistent, structured datapoints for Vulnerability, Alert Level, and Population. 

A similar structural inconsistency occurred with the GDACS GeoJSON endpoint. Standard GeoJSON responses are typically wrapped in a `FeatureCollection` array. However, the exact GDACS endpoint for single events returns a bare `Feature` object. The pipeline’s original parser failed on this assumption, resulting in empty geographic data. This was resolved by implementing a type-checking parser that dynamically handles both `Feature` and `FeatureCollection` structures, ensuring consistent coordinate extraction.

The most significant technical hurdle arose during "Deep-Tab Extraction" for the Media intelligence. Relying solely on the GDACS Media API only yielded 300-character description snippets, which are insufficient for deep Natural Language Processing (NLP). We attempted to use the `newspaper3k` library to follow the GDACS external news links and scrape the full article bodies. However, this failed severely due to HTTP 500 errors caused by aggressive anti-scraping User-Agent blocks by major news outlets (e.g., Reuters, BBC). We overcame this by building a custom scraper using `requests` and `BeautifulSoup`, injected with realistic Chrome User-Agent headers, and targeting specific semantic HTML tags (`<article>` or `<p>`). This successfully bypassed the basic blocking mechanisms, expanding our text corpus from basic snippets to tens of thousands of words per event.

2. Comparative Insights & The Evolution of Disaster Response

We compared two distinct but geographically relevant cyclones affecting highly vulnerable regions: the historical Cyclone Amphan (2020, India/Bangladesh) and the recent Cyclone Gezani (2026, Madagascar). Both regions share a mathematically identical, "Low" Coping Capacity (score 1/3) according to the INFORM Vulnerability index. 

Despite this shared vulnerability, the data reveals a stark, concerning divergence in global attention. Amphan generated a massive 61,000-word media corpus. When correlated with population exposure, it yielded a "Forgotten Crisis Index" (FCI) of 4,360, categorizing it as massively over-covered. In contrast, the recent Cyclone Gezani generated merely a 16,000-word corpus. With an FCI of just 78, it is classified as severely under-reported. 

Is the world responding faster today? From a purely temporal standpoint, yes. The Response Delta (ΔT) for Amphan—the time between the GDACS system alert and peak media volume—was 90.0 hours. For the recent Cyclone Gezani, the media peaked on the exact same day as the system alert (ΔT = 0.0 hours). Information velocity has increased significantly.

However, is media focus aligning with areas of highest vulnerability? Our intelligence indicates it is not. While the speed of reporting has improved, the allocation of attention remains disproportionately skewed. A "Low" Coping Capacity region like Madagascar (Gezani) received 55.6x less media coverage per capita than India/Bangladesh (Amphan). Furthermore, sentiment analysis reveals that coverage of Gezani was significantly more "analytical" (75% vs 67%) and less "alarmist" than Amphan. This subdued tone, combined with the lack of volume, suggests that while modern alert systems detect crises instantly, global media attention still fails to align equitably with localized humanitarian vulnerability.
